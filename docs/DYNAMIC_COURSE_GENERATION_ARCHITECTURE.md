# 动态课程生成系统架构更新

## 系统概述

动态课程生成系统是iMato平台的AI驱动教育功能模块，基于GPT-3.5 Turbo模型为不同年级和学习背景的学生生成个性化的课程设计方案。

## 架构变更

### 新增组件

#### 1. 核心服务层
```
backend/ai_service/dynamic_course.py
├── DynamicCourseGenerator     # 核心生成器类
├── StudentProfile            # 学生档案模型
├── CourseTemplate            # 课程模板模型
├── DynamicCourseRequest      # 生成请求模型
└── DynamicCourseResponse     # 生成响应模型
```

#### 2. 数据模型层
```
backend/models/dynamic_course.py
├── GeneratedCourse           # 生成课程记录表
├── CourseGenerationLog       # 生成日志表
├── StudentProfileCreate      # 学生档案创建模型
├── DynamicCourseRequestCreate # 课程请求创建模型
└── 相关响应模型...
```

#### 3. API路由层
```
backend/routes/dynamic_course_routes.py
├── POST /ai/dynamic-course          # 生成动态课程
├── GET /ai/dynamic-course/history   # 获取生成历史
├── GET /ai/dynamic-course/stats     # 获取统计信息
├── GET /ai/dynamic-course/templates # 获取模板评估
└── POST /ai/dynamic-course/backtest # 运行A/B测试
```

#### 4. 回测验证层
```
backend/services/course_backtest_service.py
├── BacktestManager              # 回测管理器
├── CourseStatistics             # 课程统计模型
├── CourseComparison             # 课程对比数据
└── 相关分析函数...
```

## 数据流向

### 课程生成流程
```
1. 用户请求 → 2. 参数验证 → 3. 模板选择 → 4. 提示词构建 
     ↓              ↓              ↓              ↓
5. OpenAI调用 → 6. 响应解析 → 7. 数据存储 → 8. 结果返回
```

### 回测验证流程
```
1. 数据收集 → 2. 统计分析 → 3. 显著性检验 → 4. 报告生成
     ↓              ↓              ↓              ↓
生成课程数据    完成率计算    t检验分析      性能报告输出
```
