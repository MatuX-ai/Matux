## 1. 产品概述

### 1.1 产品定位

MatuX 桌面端是面向 **学生** 的 AI 编程与 STEM 学习客户端，基于 Electron 框架封装 Angular Web 应用，内嵌 Python 后端服务，为大屏场景（PC/笔记本）提供沉浸式学习体验。 *(v1)*

> **角色范围**：MatuX 仅有 **学生 (student)** 一个角色。机构管理功能已解耦至独立项目 `OpenMTEduInst`。 *(v1)*

### 1.2 与移动端的关系 *(v1)*

| 维度 | 桌面端 (Electron) | 移动端 (Flutter) |
|------|-------------------|------------------|
| 定位 | 编程 + STEM 实验主力平台 | 课程学习 + 日常练习 |
| 屏幕 | 1920×1080 大屏 | 手机/平板触屏 |
| 核心场景 | 代码编写、3D 实验、创作 | 视频课程、测验、进度查看 |
| 输入方式 | 键盘 + 鼠标 | 触屏 |
| 离线能力 | 全功能离线（内嵌后端） | 缓存模式离线 |

### 1.3 核心价值 *(v1)*

| 维度 | 价值描述 |
|------|----------|
| **IDE 级编程体验** | Monaco Editor + AI 代码补全，键盘快捷键驱动 |
| **沉浸式实验** | GPU 加速的 AR/VR 实验室、数字孪生、电路仿真 |
| **离线可用** | 内嵌 Python 后端 + PWA 缓存，断网也能写代码做实验 |
| **多窗口创作** | 代码编辑、3D 预览、课程视频可拆分窗口 |
| **成果导出** | 项目文件本地保存、作品分享、学习报告生成 |

### 1.4 目标用户 *(v1)*

**唯一角色：学生 (student)**

- 年龄范围：8-18 岁（少儿编程 + 青少年 STEM）
- 设备：Windows/macOS 笔记本或台式机
- 场景：在家自学、学校机房、编程兴趣班

## 4. 技术架构 *(v1)*

### 4.1 整体架构

四层架构：Electron Shell（主进程 + 预加载 + Angular 渲染进程）→ IPC 通信 → Python 后端服务（FastAPI，localhost:8000）

### 4.2 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 桌面框架 | Electron | 28.0.0 |
| 前端框架 | Angular | 21.2.x |
| UI 组件库 | Angular Material | 21.2.x |
| 代码编辑器 | Monaco Editor | - |
| 可视化编程 | Blockly | - |
| 3D 引擎 | three.js | - |
| 图表 | ECharts / Chart.js | - |
| 后端框架 | FastAPI (Python) | 3.9+ |
| 数据库 | SQLite | - |
| 向量数据库 | ChromaDB | - |
| 嵌入模型 | BGE-small-zh / text2vec | - |
| 打包工具 | electron-builder | 24.9.1 |

### 4.3 安全模型 *(v1)*

- 主进程：Node.js 完整权限（文件读写、子进程、系统托盘、自动更新）
- IPC Bridge：`contextBridge` + 白名单频道（仅暴露文件操作、对话框、系统信息）
- 渲染进程：沙箱环境（`nodeIntegration: false`, `contextIsolation: true`, CSP, DOM Purify XSS）

### 4.4 与外部项目的关系 *(v1 + 新增)*

- **OpenMTSciEd / OpenSciEDU**（课件资源端）：
  - 开发/本地环境：API 端点 `localhost:3000/api/v1`
  - **生产环境：自动 API 连接 `https://opensciedu.matux.tech/`**
  - 学习端通过该 API 获取公共课程内容，不包含课件管理功能
  - 支持在应用内以浏览器视图装入 OpenSciEDU 知识图谱
- **OpenMTEduInst**（机构管理端）：MatuX 通过 API 获取学生课程列表和学习数据，不包含机构管理功能

## 5. 非功能需求 *(v1)*

### 5.1 性能指标

| 指标 | 目标值 |
|------|--------|
| 冷启动时间 | < 5 秒（含后端启动） |
| 热启动时间 | < 2 秒 |
| 页面切换 | < 300ms |
| 代码编辑器响应 | < 50ms 输入延迟 |
| 3D 渲染帧率 | ≥ 30fps |
| 内存占用 | < 500MB（含后端） |
| 安装包大小 | < 200MB |

### 5.2 兼容性

| 平台 | 最低版本 | 架构 |
|------|----------|------|
| Windows | Windows 10 (1809+) | x64 |
| macOS | macOS 11 (Big Sur+) | x64, arm64 |
| Linux | Ubuntu 20.04+ | x64 |

### 5.3 可靠性

- 崩溃率 < 0.1%
- Python 进程崩溃后 3 秒内自动重启
- 代码每 30 秒自动保存，异常退出可恢复
- 网络恢复后自动上传离线记录

### 5.4 国际化

- 支持简体中文 (zh-CN) 和英文 (en-US)
- 切换语言无需重启
- Blockly 积木块支持双语

### 5.5 无障碍 (a11y)

- 键盘全操作（Tab/Enter/Escape/方向键）
- 高对比度模式
- 屏幕阅读器兼容（ARIA 标签）
- 符合 WCAG 2.1 AA 标准

| 卡片入场 | `opacity + scale: 0.95 → 1`，stagger 0.1s | 列表加载 |
| 卡片悬停 | `scale: 1.05, y: -4`，spring | 可点击卡片 |
| 按钮点击 | `scale: 0.95 → 1` | 所有按钮 |
| 浮动按钮 | `scale: 0 → 1` | AI 助手入口 |
| 通知弹出 | `y: 50 → 0` | Toast |
| 模态框 | `scale: 0.9 → 1` | 对话框 |
| 进度条 | `width 过渡` | 学习进度 |

### 6.8 组件设计规范 *(v1)*

#### 统计卡片
- 圆角 `rounded-xl`，居中文本，阴影 `shadow-md`，悬停上移

#### 课程卡片
- 渐变背景 + emoji 图标，底部「开始学习」按钮，`rounded-2xl`

#### 进度任务行
- 左侧状态圆点（○/✓），中间标题+元数据，右侧进度条+百分比

#### AI 浮动按钮
- 56×56px 圆形，渐变背景 `linear-gradient(135deg, #667eea 0%, #764ba2 100%)`，`shadow-2xl`
- 右下角固定定位 `position: fixed; bottom: 48px; right: 24px`（48px 确保不与底部状态栏 28px 重叠）
- 悬停放大 `scale(1.1)`，点击展开 AI 对话面板

### 6.9 设计原则 *(v1)*

1. 品牌一致性：与 `edu-proto` 原型站保持一致的色彩、字体、圆角、阴影体系
2. 聚焦学习：界面围绕学生学习流设计，去除一切管理元素
3. 沉浸式编码：代码区采用暗色终端风格，支持全屏禅模式
4. 趣味性：emoji 图标、渐变色卡片、成就动画
5. 桌面端适配：多列网格布局（4 列快捷入口、3 列推荐课程、2 列任务+统计）
6. 响应式：窗口缩放自适应，最小 1024×768 布局不错乱
7. 暗色模式：支持亮色/暗色主题切换

### 6.10 桌面端 vs 移动端差异 *(v1)*

| 维度 | 移动端 | 桌面端 |

| 快捷入口 | 2×2 网格 (4 个) | 1×4 网格 (4 个大卡片) |
| 课程列表 | 单列垂直滚动 | 3 列网格 |
| 任务列表 | 纵向列表 | 左任务+右统计双栏 |
| 代码编辑器 | 全屏独占 | 左题目+右代码分栏 |
| AI 助手 | 全屏模态框 | 右侧浮窗面板 |
| 底部状态 | 无 | 状态栏（在线状态+版本号） |

## 9. 风险与依赖 *(v1)*

### 9.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| Python 环境依赖复杂 | 安装包过大、启动失败 | 中 | PyInstaller 打包为独立可执行文件 |
| 代码执行沙箱安全 | 恶意代码风险 | 中 | Docker 隔离 / 资源限制 |

| AI 教师回答质量 | 不准确/不当回答 | 中 | Prompt 工程 + 内容审核层 + 学生反馈 |
