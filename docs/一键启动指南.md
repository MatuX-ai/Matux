# iMato 项目一键启动指南

## 🚀 快速启动方式

### 方式一：VSCode 任务启动（推荐）

1. **打开 VSCode 工作区**
   - 打开 `G:\iMato` 文件夹

2. **运行一键启动任务**
   - 按 `Ctrl+Shift+P` 打开命令面板
   - 输入 `Tasks: Run Task`
   - 选择 `Start All Services`
   
   或者使用快捷键：
   - `Ctrl+Shift+B` - 运行默认构建任务（Start All Services）

3. **等待服务启动完成**
   - Redis 会自动启动（如果有安装）
   - 后端服务会在 http://localhost:8000 启动
   - 前端服务会在 http://localhost:4200 启动

### 方式二：命令行脚本

```bash
# Windows PowerShell
.\start-all.bat

# 或者分别启动
.\start-backend.bat    # 后端
.\start-frontend.bat   # 前端
```

## 📋 可用的 VSCode 任务

| 任务名称 | 说明 | 快捷键 |
|---------|------|--------|
| Start All Services | 同时启动 Redis、后端、前端 | Ctrl+Shift+B |
| Start Backend Only | 仅启动后端服务 | - |
| Start Frontend Only | 仅启动前端服务 | - |
| Kill All Services | 强制关闭所有服务 | - |

## 🔍 查看服务状态

启动后可以访问：
- **前端界面**: http://localhost:4200
- **后端 API**: http://localhost:8000
- **API 文档**: http://localhost:8000/docs

## ⚙️ 服务配置说明

### Redis
- 路径：`G:\Redis\redis-server.exe`
- 配置文件：`G:\iMato\backend\redis\redis.windows.conf`
- 端口：6379
- 如果 Redis 未安装，后端会使用内存存储模式

### 后端 (FastAPI + Python)
- 启动脚本：`backend/run.py` 或 `backend/main.py`
- Python 路径：`G:\Python312\python.exe`
- 端口：8000
- 自动检测并启动 Redis 和 Neo4j

### 前端 (Angular)
- 命令：`npm start`
- 端口：4200
- 支持热重载

## 🛠️ 常见问题

### 1. 端口被占用
如果提示端口被占用，可以：
- 运行 `Kill All Services` 任务清理进程
- 或者手动关闭占用端口的程序

### 2. Redis 未启动
- 检查 `G:\Redis` 是否存在
- 如果没有 Redis，后端会自动降级到内存模式
- 不影响基本功能使用

### 3. Neo4j 未启动
- 需要在 Neo4j Desktop 中手动启动数据库实例
- 或者使用 Docker: `docker run -d -p 7474:7474 -p 7687:7687 neo4j:latest`
- 没有 Neo4j 时，相关功能会降级

### 4. 前端编译慢
- 首次启动会较慢，因为需要编译所有模块
- 后续启动会快很多（增量编译）
- 耐心等待 `Compiled successfully` 提示

## 📝 自定义配置

如需修改启动配置，编辑 `.vscode\tasks.json` 文件：
- 修改端口号
- 调整启动顺序
- 添加新的服务

## 💡 最佳实践

1. **推荐开发模式**：使用 VSCode 任务启动，可以实时看到各服务的输出日志
2. **停止服务**：直接关闭 VSCode 终端窗口即可
3. **重启服务**：先运行 `Kill All Services`，再运行 `Start All Services`
