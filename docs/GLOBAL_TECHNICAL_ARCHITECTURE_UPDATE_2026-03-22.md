## 🔧 技术规范更新

### 1. 前端开发强制规范

#### 类型安全 (零容忍)
```typescript
// ❌ 禁止
const data: any = await service.getData();

// ✅ 必须
interface ApiResponse<T> {
  success: boolean;
  data: T;
  message?: string;
}
const response: ApiResponse<Course[]> = await courseService.getAll();
```

#### 命名规范 (强制统一)
```typescript
// ❌ 禁止
interface UserProfile { firstName: string; lastName: string; }

// ✅ 必须
interface UserProfile { first_name: string; last_name: string; }
```

#### SCSS预算控制
- **单个文件**: ≤ 4KB (4096字节)
- **超标处理**: 必须拆分到共享样式文件
- **共享样式库**: `_buttons.scss`, `_cards.scss`, `_forms.scss`等

#### 组件架构
- **Smart组件**: 处理业务逻辑和状态管理
- **Dumb组件**: 纯展示，无状态依赖
- **通信方式**: @Input/@Output + 服务注入

### 2. 后端API规范

#### 响应格式标准化
```typescript
// 标准响应
interface ApiResponse<T> {
  success: boolean;
  data: T;
  message?: string;
  timestamp: string;
}

// 分页响应  
interface PaginatedResponse<T> {
  items: T[];
  total: number;
  page: number;
  page_size: number;
  total_pages: number;
}
```

#### 错误处理协议
```http
# 标准错误响应
HTTP/1.1 400 Bad Request
Content-Type: application/json

{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "请求参数验证失败",
    "details": {
      "field": "email",
      "reason": "格式不正确"
    }
  },
  "timestamp": "2026-03-22T10:30:00Z"
}
```
