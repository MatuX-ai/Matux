## 二、根因分析

### 2.1 系统性问题

`_stem-theme.scss` 现有 light 变体作为正文/图标色 vs 白色/浅色背景，**全部不达 WCAG AA 4.5:1**：

| 令牌 | 色值 | vs #fff | 评级 |
|------|------|---------|------|
| `$stem-primary` | #059669 | 3.77:1 | ✗ AA |
| `$stem-success` | #22c55e | 2.28:1 | ✗ AA |
| `$stem-warning` | #f59e0b | 2.15:1 | ✗ AA |
| `$stem-error` | #ef4444 | 3.77:1 | ✗ AA |
| `$stem-secondary` | #0ea5e9 | 2.77:1 | ✗ AA |

### 2.2 渐变底端退化

`linear-gradient(135deg, $stem-primary 0%, $stem-secondary 100%)` 等组合，**底端色** vs 白字对比度往往 <3:1（不达 WCAG 1.4.11 非文本 3:1）。

### 2.3 透明度叠加

`opacity: 0.5` / `opacity: 0.8` 等全局透明度，会显著降低对比度（即便深色背景上白字 opacity 0.5 仍 <3:1）。

### 2.4 嵌套状态

`.skill-item.locked { opacity: 0.5 }` 这种"父级透明 + 子级色字"的反模式，复合后对比度骤降。

### 2.5 inline [style.color] 模板

`getScoreColor()`、`moodOptions[]` 等 TS 内联颜色字面量，**绕开设计令牌**，独立于 stylelint 防护。

## 三、修复方案

### 3.1 设计令牌扩展（_stem-theme.scss）

新增 6 个 **dark 变体令牌**，所有 dark 变体 vs #fff 均 ≥ 4.5:1 AA：

```scss
$stem-primary-dark: #047857;        // Emerald-700，vs #fff 5.18:1 AA
$stem-secondary-dark: #0284c7;      // Sky-600，vs #fff 4.62:1 AA
$stem-accent-dark: #0e7490;         // Cyan-700，vs #fff 5.51:1 AA
$stem-success-dark: #15803d;        // Green-700，vs #fff 5.13:1 AA
$stem-warning-dark: #b45309;        // Amber-700，vs #fff 4.62:1 AA
$stem-warning-dark-emphasis: #92400e; // Amber-800，vs #fff 7.81:1 AAA
$stem-error-dark: #dc2626;          // Red-600，vs #fff 4.83:1 AA
$stem-accent-blue-dark: #2563eb;    // Blue-600，vs #fff 5.16:1 AA
```

> 注：实际生效 8 个（含原有 `$stem-primary-dark` 和 `$stem-secondary-dark`），覆盖所有 light 变体的 dark 配对。

### 3.2 统一修复模式

| 修复模式 | 应用处数 | 原理 |
|---------|---------|------|
| `$stem-X` → `$stem-X-dark` | ~50 | light 令牌直接替换为 dark 变体（vs #fff 4.5:1+） |
| `rgba($stem-X, 0.05~0.3)` → `rgba($stem-X-dark, ...)` | ~10 | 浅色背景应用 dark 半透明版 |
| `linear-gradient` 底端换 dark | ~8 | 渐变底端避免退化到不达 3:1 |
| 移除 `opacity: 0.X` 改用 `rgba` 显式色 | 2 | token-dashboard 白字 + opacity 0.8 改 rgba 颜色 |
| 移除 `opacity: 0.5` 改用明确色值 | 1 | learning-profile `.skill-item.locked` |
| `var(--color-error)` / `var(--color-accent)` → 显式 hex | ~6 | 跨主题色值稳定保证 4.5:1+ |
| `var(--mat-text-secondary)` → `var(--color-text-secondary)` | ~5 | 适配 Material 21 token 命名 |
| inline hex → dark hex（#059669→#047857 等） | 14 | ECharts / scoreColor / moodOptions 升级 |

### 3.3 stylelint 防护规则扩展（.stylelintrc.js）

在 `declaration-property-value-disallowed-list` 规则中扩展 **5 个 SCSS 变量** + **9 个 hex 值** 黑名单：

```javascript
color: [
  // 原有
  '/var\\(\\s*--color-text-disabled/',
  '/var\\(\\s*--color-gray-400\\b/',

  // 新增 SCSS 变量（negative lookahead 排除 -dark 变体）
  '/\\$stem-primary(?!-dark)\\b/',
  '/\\$stem-success(?!-dark)\\b/',
  '/\\$stem-warning(?!-dark)\\b/',
  '/\\$stem-error(?!-dark)\\b/',
  '/\\$stem-secondary(?!-dark)\\b/',

  // 新增硬编码 hex（vs #fff <4.5:1）
  '/^#22c55e$/',  // green-500
  '/^#f59e0b$/',  // amber-500
  '/^#ef4444$/',  // red-500
  '/^#0ea5e9$/',  // sky-500
  '/^#94a3b8$/',  // slate-400
  '/^#a8a29e$/',  // stone-400
  '/^#86efac$/',  // green-300
  '/^#a1a1aa$/',  // zinc-400
  '/^#f97316$/',  // orange-500
  '/^#3b82f6$/',  // blue-500
],
```

> negative lookahead `(?!\-dark)` 确保规则智能识别 dark 变体不误报。

## 七、设计原则沉淀

### 7.1 色彩选用规则

1. **正文文字**（12-18px、常规字重）：必须 ≥ 4.5:1 AA
2. **大号文字**（≥18px 或 ≥14px 加粗）：必须 ≥ 3:1 AA
3. **图标/UI 控件**：必须 ≥ 3:1（WCAG 1.4.11）
4. **装饰元素**：不强制，但建议 ≥ 1.5:1 防止视觉干扰

### 7.2 令牌使用规则

| 场景 | 推荐令牌 | 反例 |
|------|---------|------|
| 正文/图标（vs 白底） | `$stem-X-dark` 系列 | `$stem-X` 系列 |
| 背景色（用作容器） | `$stem-X` 系列 | `$stem-X-dark`（过深） |
| 半透明背景/边框 | `rgba($stem-X-dark, 0.X)` | `rgba($stem-X, 0.X)`（可能不达 3:1） |
| 渐变两端 | `dark → dark` 或 `dark → light` | `light → light`（底端退化） |
| 父级 `opacity` | 禁止（破坏子级对比度） | `opacity: 0.5`（反模式） |

### 7.3 内联色值规则

1. 禁止在 TS 中硬编码 hex 颜色字面量
2. 必须从设计令牌（`_stem-theme.scss`）导入使用
3. ECharts 等第三方库需要内联颜色时，统一从令牌对象读取

### 7.4 stylelint 防护规则

新代码提交必须通过 stylelint `declaration-property-value-disallowed-list` 规则。negative lookahead `(?!\-dark)` 已确保 dark 变体不被误报。
